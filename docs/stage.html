Stage chez Efficience Digitale — Développement Back-Office Qobook

Création d’un back-office complet pour l’application mobile Qobook, incluant scripts Node.js, outils internes et interface Retool.

Présentation du stage

Ce stage de deuxième année de BUT Informatique s’est déroulé au sein de l’agence Efficience Digitale, spécialisée dans le développement web et la création de sites pour divers clients. L’un de leurs projets internes, Qobook, est une application mobile regroupant plus de 4 millions de livres, permettant d’accéder à des contenus culturels, des extraits ou encore des espaces thématiques.

Ma mission principale a été de concevoir et développer une large partie du back-office permettant de créer, modifier et gérer des espaces personnalisés dans l’application. Ce back-office devait être utilisable par l’équipe non technique et permettre une gestion entièrement autonome des espaces.

Mission principale : Développement du Back-Office de Qobook

Au début du stage, la création d’un espace personnalisé nécessitait l’intervention d’un développeur pour insérer manuellement les données dans la base Supabase. L’objectif était donc d’automatiser totalement cette création via :

  • Un site interne facilitant la création de fichiers CSV structurés
  • Des scripts Node.js capables de lire, parser et insérer ces données
  • Un serveur Express pour exécuter dynamiquement ces scripts
  • Une intégration finale complète dans Retool pour créer un vrai back-office professionnel

1) Prototype « from scratch »

J’ai développé un premier prototype autonome comprenant :

  • Un site HTML/JS permettant de générer les fichiers CSV nécessaires (espaces, thèmes, livres, carrousels)
  • Des scripts Node.js (JavaScript côté serveur) pour insérer les données dans la base Supabase
  • Un serveur Express permettant d’exécuter les scripts dynamiquement à partir du navigateur
  • Un script dédié à la modification d’espaces existants

Ce prototype m’a permis d’apprendre JavaScript (front + Node.js), de manipuler des APIs Supabase, de gérer des fichiers CSV et de comprendre l’architecture complexe de la base de données de Qobook.

2) Version finale sur Retool

Lorsque le prototype a été validé, j’ai migré toute la solution vers Retool, un outil low-code basé sur React. L’interface finale permet :

  • De lister tous les espaces existants
  • De naviguer entre les pages d’informations, de modification et de création
  • D’effectuer toutes les requêtes via une ressource REST API sécurisée
  • D’ajouter dynamiquement des carrousels via un TabbedContainer
  • De modifier des tables liées (thèmes, livres, composants visuels) via des tableaux éditables

Le back-office final permet désormais à toute l’équipe de gérer les espaces personnalisés de façon simple, rapide et sans intervention technique.

Technologies utilisées

JavaScript Node.js Express.js Retool API REST Supabase CSV React (indirect)

L’ensemble du projet repose sur une architecture orientée scripts Node.js, requêtes API et interfaces internes low-code. Ce mélange m’a permis de découvrir de nouveaux outils et de renforcer mes compétences en développement web moderne.

Compétences du BUT Informatique mobilisées

Compétence 1 : Développer une application

  • Création de scripts Node.js pour lire et insérer les données
  • Développement d’une interface interne en JavaScript
  • Mise en place d’un serveur Express pour exécuter les scripts

Compétence 2 : Optimiser des applications

  • Analyse d’une base de données complexe et existante
  • Structuration des CSV pour éviter la redondance et faciliter l’insertion
  • Réduction des manipulations manuelles grâce à l’automatisation

Compétence 4 : Gérer des données

  • Requêtes API Supabase (insert, update, delete)
  • Manipulation de fichiers CSV et transformation en objets JS
  • Modélisation pour la gestion des thèmes, livres et carrousels

Compétence 5 : Conduire un projet

  • Analyse des besoins métier avec l’équipe
  • Choix techniques argumentés (JS, Node, Retool)
  • Livraison étape par étape : prototype → version stable

Compétence 6 : Travailler en équipe

  • Collaboration avec le lead dev et un alternant
  • Prises d’initiative pour améliorer l’ergonomie des outils internes
  • Communication régulière avec les équipes non techniques
