Very Bad Split

Refactoring d'une application web de partage de dépenses entre amis

Description du projet

Very Bad Split est un projet de refactoring d'une application web existante permettant de gérer et partager les dépenses entre amis lors de voyages, collocations ou événements. L'objectif principal était de moderniser l'architecture technique et d'améliorer l'expérience utilisateur en appliquant les concepts avancés appris lors du 4ème semestre de BUT Informatique.

Le projet comportait plusieurs défis techniques :

  • Refactoriser une base de code PHP traditionnelle vers une architecture Symfony
  • Implémenter un système de routage robuste et sécurisé
  • Créer une API RESTful pour les interactions client-serveur
  • Utiliser Twig pour la gestion des templates front-end
  • Intégrer des fonctionnalités réactives avec JavaScript moderne
  • Améliorer la gestion des données et la sécurité de l'application

L'application permet aux utilisateurs de :

  • Créer des groupes pour gérer les dépenses collectives
  • Ajouter des dépenses et les répartir équitablement ou selon des critères personnalisés
  • Visualiser en temps réel qui doit de l'argent à qui et combien
  • Marquer les dettes comme remboursées et garder un historique des transactions
  • Exporter les bilans de dépenses dans différents formats

Ce projet a été réalisé dans le cadre du 4ème semestre de BUT Informatique, mettant en pratique les connaissances acquises en développement web avancé et en architecture d'applications.

Technologies utilisées

Symfony PHP Twig JavaScript API REST MySQL Normalisation BD Git

La refonte technique de Very Bad Split s'est appuyée sur des technologies modernes :

  • Symfony : Framework PHP moderne utilisé pour structurer l'application selon le modèle MVC, permettant une séparation claire des responsabilités et facilitant la maintenance du code.
  • Système de routage Symfony : Mise en place d'un routage avancé pour une gestion sécurisée des URLs et des requêtes, avec annotations pour simplifier la configuration.
  • Twig : Moteur de templates utilisé pour la couche de présentation, offrant une syntaxe élégante et des fonctionnalités avancées comme l'héritage de templates et les macros.
  • API RESTful : Développement d'endpoints API pour permettre une communication fluide entre le front-end et le back-end, en utilisant des formats JSON standardisés.
  • JavaScript réactif : Utilisation de JavaScript moderne pour créer des interactions dynamiques côté client, avec mise à jour en temps réel des calculs de dépenses sans rechargement de page.
  • Normalisation de base de données : Application des formes normales (jusqu'à 3NF) pour optimiser la structure de la base de données, éliminer les redondances et garantir l'intégrité des données. Mise en place de clés étrangères et de contraintes d'intégrité pour assurer la cohérence des relations entre utilisateurs, groupes, dépenses et remboursements.
  • MySQL : Base de données relationnelle pour stocker les informations sur les groupes, utilisateurs, dépenses et remboursements.

L'architecture du projet suit les meilleures pratiques de développement web moderne, avec une séparation claire entre le back-end (API Symfony) et le front-end (Twig + JavaScript), permettant une maintenance facilitée et une évolution progressive des fonctionnalités.

Compétence 2 : Optimiser des applications

  • AC 2.1 : Analyser un problème avec méthode → Analyse de l'ancienne base de code pour identifier les points faibles et planifier le refactoring.
